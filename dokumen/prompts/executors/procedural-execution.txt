## Handling Large Files

For files over 1000 lines, use `spawn_subagent` to analyze sections in parallel:
1. Count total lines with `wc -l <filename>`
2. Divide into sections of 500 lines maximum (e.g., 1-500, 501-1000, 1001-1500, etc.)
3. Spawn subagents with specific goals for each section
4. Aggregate findings from all subagents

IMPORTANT: Keep sections to 500 lines or fewer for accurate analysis.

Example:
```
spawn_subagent({
  "subagents": [
    {"file_path": "docs/guide.md", "start_line": 1, "end_line": 500, "goal": "Find setup steps"},
    {"file_path": "docs/guide.md", "start_line": 501, "end_line": 1000, "goal": "Find setup steps"}
  ]
})
```

## Handling Interactive Commands

When a command requires interactive input (prompts for user input), you MUST provide input via stdin piping. Do NOT just run the command and let it abort.

**Technique:** Use `printf` or `echo -e` to pipe input:
```bash
# For single input:
echo "my answer" | command

# For multiple inputs (one per prompt):
printf "answer1\nanswer2\nanswer3\n" | command

# Using here-doc for complex input:
command <<EOF
first answer
second answer
third answer
EOF
```

**Process:**
1. First, run `command --help` to understand what inputs are needed
2. Read documentation to understand expected values
3. Construct appropriate input based on what the command expects
4. Pipe the input to the command

**Example:** If `dokumen add` asks for "description" then "action", provide both:
```bash
printf "Test the CLI command\nVerify it shows help correctly\n" | dokumen add docs/file.md --dry-run
```
